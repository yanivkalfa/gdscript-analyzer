# GODOT-SYNC — The Rule File for Tracking Godot & Propagating API Changes

> **The owner's #1 priority.** This is the **rule file**: the precise, automatable process for tracking *every* Godot release and propagating engine-API changes into the analyzer. It is written to be implemented as automation **and** followed by hand. Every constant (URL, regex, command, path) is concrete; the GitHub Action is sketched in full.

**Canonical inputs:** [`research/03-godot-api-sync.md`](research/03-godot-api-sync.md) (the schema, sources, what's missing, version detection — the PRIMARY evidence) and [`research/07-ecosystem-and-release-tooling.md`](research/07-ecosystem-and-release-tooling.md) §8 (the Action mechanics). Obeys the data model in [`01-ARCHITECTURE.md`](01-ARCHITECTURE.md) §5 and the `gdscript-api` crate contract in §1. The `cargo xtask codegen-api` step is owned by Phase 0 ([`PHASE-0-ECOSYSTEM-AND-TOOLING.md`](PHASE-0-ECOSYSTEM-AND-TOOLING.md)).

## Purpose

Keep the analyzer's **engine knowledge correct as Godot evolves.** The `gdscript-api` crate is the analyzer's model of "what Godot offers" — every built-in class, method, property, signal, enum, constant, singleton, utility function, and Variant type, plus the hand-authored GDScript language layer the engine dump omits. This crate is **generated, never hand-maintained** (`01-ARCHITECTURE.md` §5): `cargo xtask codegen-api` lowers a vendored `extension_api.json` + Godot doc XML into the crate's data.

The risk this document neutralizes: Godot ships a new minor (`4.6`, `4.7`, …), adds/renames/removes API, and our analyzer silently reports stale completions, wrong hovers, and false diagnostics. The cure is a **deterministic, machine-readable, re-runnable sync pipeline** triggered by every release, producing a reviewable PR with a classified API delta.

**The three rules that make this work**

1. **Source of truth = `extension_api.json`** — the `godot --headless --dump-extension-api` output, the *same artifact* `gdext`, godot-cpp, and the C#/.NET bindings consume. We never scrape docs or HTML for the API itself.
2. **The dump is incomplete by design.** It is an *engine/native* (ClassDB + Variant) dump only. The **GDScript language layer** (keywords, annotations, builtins, the `@GDScript`/`@GlobalScope` pseudo-classes) lives only in doc XML or nowhere — we hand-author it and track it on a **separate checklist** (see [The hand-authored GDScript layer sync](#the-hand-authored-gdscript-layer-sync-separate-checklist)).
3. **The only fully machine-readable break signal is a diff of two dumps**, with each method's **`hash`** change = a signature change. Human changelogs are advisory, not parseable.

## What "Godot's API" means to us — the inputs we track

"Godot's API" to the analyzer is **three layers**, each with its own source and its own freshness obligation:

1. **The engine/native API** — `extension_api.json`. Authoritative for classes, builtins, enums, constants, singletons, utility functions, signatures (via `hash`). Machine-readable. Diffable. **Auto-synced** (the pipeline below).
2. **Hover/doc text** — per-class doc XML (`doc/classes/*.xml` + module doc classes), BBCode → Markdown. Strictly more complete than the JSON `--with-docs` variant (it carries `<tutorials>`, per-constant prose, and the `<annotation>` elements). **Auto-fetched** at the same tag, regenerated by `xtask`.
3. **The hand-authored GDScript layer** — everything the dump omits: GDScript keywords, the 36 annotations, GDScript builtins (`preload`/`load`/`range`/`len`/…), and the `@GDScript`/`@GlobalScope` pseudo-classes. Sourced from GDScript module docs/source. **Manually maintained** against a per-minor checklist.

> Project-local symbols (autoloads, `class_name` globals, `[input]` actions, `[global_group]`) are **not** part of "Godot's API" and are **never** bundled — they come from the consumer's `project.godot` resolved at runtime ([`research/03`](research/03-godot-api-sync.md) §6). This doc does not cover them.

### Input map

| Input | What it feeds | Source URL / pattern | Sync mode |
|---|---|---|---|
| `extension_api.json` (no-docs) | engine classes, builtins, enums, constants, singletons, utility fns, signatures (`hash`) | `https://raw.githubusercontent.com/godotengine/godot-cpp/godot-{ver}-stable/gdextension/extension_api.json` | **auto** (pipeline) |
| `extension_api.json` (secondary) | cross-check / fallback source for the same | `https://raw.githubusercontent.com/godot-rust/godot4-prebuilt/{ver}/res/extension_api.json` | **auto** (fallback) |
| Engine doc XML (`doc/classes/*.xml`) | hover/doc text for engine classes; member/signal/constant prose; tutorials | godot repo @ tag `{ver}-stable`, path `doc/classes/*.xml` | **auto** (pipeline) |
| `@GlobalScope.xml` | docs for global enums/constants/utility-fn prose | `https://raw.githubusercontent.com/godotengine/godot/{ver}-stable/doc/classes/@GlobalScope.xml` | **auto** + manual review |
| `@GDScript.xml` | **annotations** + GDScript builtins docs (exists nowhere else) | `https://raw.githubusercontent.com/godotengine/godot/{ver}-stable/modules/gdscript/doc_classes/@GDScript.xml` | **auto-fetch + manual author** |
| GDScript keyword set | tokenizer/keyword completions | godot `modules/gdscript/gdscript_tokenizer.*` / GDScript reference | **manual** (per-minor checklist) |
| Annotation list (36) | `@export*`/`@onready`/`@tool`/`@rpc`/… completions + checks | godot `modules/gdscript/gdscript.cpp` annotation registrations + `@GDScript.xml` | **manual** (per-minor checklist) |
| GDScript warning list | the GDScript warning diagnostics (separate concern) | godot `modules/gdscript/gdscript_warning.h` | **manual** (per-minor checklist) |
| GDScript builtins | `preload`/`load`/`range`/`len`/`char`/`ord`/… | `@GDScript.xml` `<methods>` | **manual** (per-minor checklist) |
| Release feed | *detection* — "a new Godot exists" | GitHub releases/tags API (below) | **auto** (trigger) |

Provenance for every fetched artifact is recorded in a `meta.json` beside the snapshot (source URL, `header.version_full_name`, `header.precision`, fetch date).

## The `extension_api.json` schema we depend on

Concise reference; full field-by-field treatment in [`research/03`](research/03-godot-api-sync.md) §1. Top-level keys (order as in the real file; verified against godot-cpp@master = Godot v4.7.stable, single precision, ≈6.7 MB):

| Key | Shape | What we consume |
|---|---|---|
| `header` | `{version_major, version_minor, version_patch, version_status, version_build, version_full_name, precision}` | **the authoritative version + precision.** `version_status ∈ stable\|beta\|rc\|dev`; `precision ∈ single\|double`. Drives bundling + hash scope. |
| `builtin_class_sizes` | 4 build configs (`float_32/float_64/double_32/double_64`) → per-type sizes | mostly ignored (ABI detail); presence verified. |
| `builtin_class_member_offsets` | same 4 configs → per-member byte offsets | ignored (ABI detail). Extra key vs. naive spec. |
| `global_constants` | `[{name, value, is_bitfield}]` (non-empty in 4.7) | global constants in scope. |
| `global_enums` | `[{name, is_bitfield, values:[{name,value}]}]` | global enums (e.g. `Side`, `Key`). |
| `utility_functions` | `[{name, return_type?, category, is_vararg, hash, arguments[]}]` (`category ∈ math\|general`) | `@GlobalScope` functions (`print`, `sin`, `typeof`, …). **No** `is_const/is_static/is_virtual`. |
| `builtin_classes` | 38 Variant types: `{name, members, constants, operators, methods, constructors, enums, has_destructor, …}` | `Vector2`/`String`/`Dictionary`/… members, operators, methods. |
| `classes` | ~1036 `Object`-derived: `{name, is_refcounted, is_instantiable, inherits, api_type, constants, enums, methods, signals, properties}` (`api_type ∈ core\|editor`) | the engine class model + inheritance chain. |
| `singletons` | `[{name, type}]` | `Input`/`OS`/`Engine`/… (`type` mirrors `name`). |
| `native_structures` | `[{name, format}]` (semicolon-delimited C-struct string) | rarely surfaced; kept for completeness. |

**Per-method flags** (load-bearing — these *are* the semantics):

- `is_vararg` — variadic; suppress arity checks.
- `is_const` — const method (callable on const/ro receivers).
- `is_static` — class-level, no instance.
- `is_virtual` — overridable; may omit `hash`; **Godot 4.4+** adds `is_required`.
- **`hash`** — computed from the method's *full signature*. **A changed `hash` = a changed signature.** Valid **only** within a `(version, precision)` tuple — double-precision builds yield different hashes. This is the single deterministic signature-change signal we diff on.

> **Two-stage parse (mirror godot-rust):** a permissive raw-JSON deserialization layer (forward-compatible with unknown keys), then a normalized internal model the resolver queries. Never bind the resolver to the raw JSON shape. Implemented inside `cargo xtask codegen-api`.

## The SYNC PIPELINE

The core deliverable: a numbered, automatable runbook. Two actors — a **CI bot** (steps 1–6) and a **maintainer** (step 7). Every step is idempotent and re-runnable. Constants are pinned in a checked-in `godot-sync.toml` rule file (below) so the runbook and the Action read the same values.

### Rule-file constants (`godot-sync.toml`, checked into the repo root)

```toml
# godot-sync rules — single source of truth for the sync job and the manual runbook.
[detect]
releases_latest = "https://api.github.com/repos/godotengine/godot/releases/latest"
tags_api        = "https://api.github.com/repos/godotengine/godot/tags"
prerelease_api  = "https://api.github.com/repos/godotengine/godot-builds/releases" # dev/beta/rc (opt-in)
tag_regex       = '^\d+\.\d+(?:\.\d+)?-stable$'   # stable only; -dev/-beta/-rc ignored unless opted in

[fetch.api_json]
primary   = "https://raw.githubusercontent.com/godotengine/godot-cpp/godot-{ver}-stable/gdextension/extension_api.json"
branch    = "https://raw.githubusercontent.com/godotengine/godot-cpp/{minor}/gdextension/extension_api.json"
secondary = "https://raw.githubusercontent.com/godot-rust/godot4-prebuilt/{ver}/res/extension_api.json"

[fetch.doc_xml]
repo        = "https://github.com/godotengine/godot"            # checkout/tarball at tag {ver}-stable
classes     = "doc/classes/*.xml"
gdscript    = "modules/gdscript/doc_classes/@GDScript.xml"
globalscope = "doc/classes/@GlobalScope.xml"

[fetch]
generate_fallback = "godot --headless --dump-extension-api --dump-extension-api-with-docs --dump-gdextension-interface"

[bundle]
supported_minors = ["4.0","4.1","4.2","4.3","4.4","4.5","4.6","4.7"]
default_minor    = "4.7"      # newest bundled; the fallback when a project's version is unknown
hash_scope       = "single"   # method hashes are valid only per (version, precision)

[paths]
vendor_root   = "vendor/godot"                          # one subdir per bundled minor
snapshot_dir  = "vendor/godot/{minor}"                  # extension_api.json + doc/ + meta.json
deltas_dir    = "vendor/godot/deltas"                   # machine + human diff artifacts
```

### 1. DETECT a new Godot release

- **Trigger event:** the scheduled Action (daily cron) **or** a manual `workflow_dispatch` with a `godot_tag` override.
- **Resolve the latest stable tag** (Godot publishes many `-dev/-beta/-rc` prereleases, so filter):
  ```sh
  gh release list -R godotengine/godot --exclude-pre-releases --exclude-drafts \
    --limit 1 --json tagName --jq '.[0].tagName'
  # or: gh api repos/godotengine/godot/releases/latest --jq .tag_name
  ```
- **Validate** the tag against `detect.tag_regex` = `^\d+\.\d+(?:\.\d+)?-stable$`. Reject anything else.
- **Opt-in pre-release track:** to track `-dev/-beta/-rc`, point at `godot-builds` (`detect.prerelease_api`) and widen the regex. **Default = stable-only.**
- **New-minor gate:** if the detected `{minor}` is not in `bundle.supported_minors`, open a **tracking issue** ("Bundle Godot {minor}?") and proceed to fetch + diff anyway — adding a minor is a maintainer decision (step 7), but the delta is still useful.

### 2. OBTAIN the new `extension_api.json` + doc XML

Try sources in order; the first that returns the tag wins. **godot-cpp tags trail Godot stable slightly**, so fallbacks matter:

1. `fetch.api_json.primary` — godot-cpp at tag `godot-{ver}-stable` (exact-patch reproducible).
2. `fetch.api_json.secondary` — godot4-prebuilt branch `{ver}` (earliest prebuilt is `4.1.4`).
3. `fetch.api_json.branch` — godot-cpp branch `{minor}` (latest patch of the line; or `master` for the newest).
4. **Last resort — generate it.** Download the editor binary (`Godot_v{ver}-stable_<platform>.zip` from GitHub releases or `https://downloads.godotengine.org/`), then run, from a scratch dir (outputs land in the **current working directory**):
   ```sh
   godot --headless --dump-extension-api --dump-extension-api-with-docs --dump-gdextension-interface
   # yields: extension_api.json, extension_api.json (with docs), gdextension_interface.h
   ```
   (`--dump-extension-api` requires an *editor* build; `--headless` uses dummy drivers, no window. `--with-docs` exists since Godot 4.2.)

Then **fetch the matching doc XML** at tag `{ver}-stable` — `doc/classes/*.xml` + `@GlobalScope.xml` (in `doc/classes/`) + `@GDScript.xml` (in `modules/gdscript/doc_classes/`, **not** `doc/classes/`). Write everything into `vendor/godot/{minor}/` and record provenance in `meta.json`.

### 3. DIFF against the currently-vendored version

Structural JSON diff, modeled on prior art **[`mhilbrunner/gdapi-diff`](https://github.com/mhilbrunner/gdapi-diff)** ("Compares the API JSON dump of different Godot Engine versions"). Compare **within the same `precision`** (hashes are precision-scoped). Categorize every change:

- **classes** — added / removed / `inherits` changed / `api_type` changed.
- **methods** — added / removed; **`hash` change ⇒ signature change** (the key signal); virtual ↔ non-virtual; `is_required` flip (4.4+).
- **properties** — added / removed / type / setter-getter changed.
- **signals** — added / removed / argument-list changed.
- **enums & constants** — added / removed / value changed / `is_bitfield` flip.
- **singletons** — added / removed.
- **utility functions** — added / removed; `hash` change.

**Classification:** every change is tagged **Added** (additive, safe), **Removed** (consumer-visible break risk), **Changed-signature** (`hash` delta — break risk), or **Deprecated** (still present but flagged upstream).

**Two output artifacts** (both written under `bundle.paths.deltas_dir`):

1. **Machine-readable API-delta** — `deltas/{old}__{new}.json`: a structured object keyed by category, each entry `{kind: added|removed|changed, symbol, before?, after?}`. Drives tests + the breaking/additive classification.
2. **Human changelog** — `deltas/{old}__{new}.md`: Keep-a-Changelog-style headings (Added / Changed / Removed / Deprecated), one bullet per symbol, breaking items first. Goes into the PR body.

### 4. REGENERATE the `gdscript-api` artifact

- Run **`cargo xtask codegen-api`** — the two-stage lower (permissive raw JSON → normalized model), tagging each symbol with **`since_api` / `before_api`** availability gates (the godot-rust pattern: a symbol only exists when the targeted API supports it).
- Regenerate the **doc-XML hover index** — BBCode → Markdown (`[b]→**`, `[code]→backtick`, `[codeblocks]/[gdscript]→fenced ```gdscript```, cross-refs `[ClassName]`/`[method X]`/`[constant X]`/`[param x]` → backticked/linked symbols, `$DOCS_URL → https://docs.godotengine.org/en/{ver}`; strip unhandled tags so no literal `[...]` reaches hover). Include the GDScript-layer docs (`@GDScript.xml` annotations/builtins).
- **If adding a new minor:** bump `bundle.supported_minors` (and, if it's the newest stable, `bundle.default_minor`). This is staged by the bot but **confirmed by the maintainer** in step 7.

### 5. TEST against golden fixtures

- Run the analyzer suite against the new snapshot — parse/inference/hover/completion fixtures that **exercise engine symbols** (`fixtures/` per `01-ARCHITECTURE.md` §8).
- **Golden symbol assertions** that must always hold: e.g. `Node.add_child` resolves, `@export` is a known annotation, `preload` is a known builtin, `Vector2 + Vector2 → Vector2`, `Input` is a singleton. A removal that breaks one of these **fails the job** and forces maintainer attention.
- **Fail on unexpected deltas:** if the machine API-delta contains **Removed** or **Changed-signature** entries not present in an allow-listed expected-set, mark the PR `breaking` and require review (do not auto-merge).

### 6. SURFACE — open the PR

- Open/update a PR (stable branch `chore/godot-api-sync` ⇒ idempotent — re-runs update the same PR) containing: the new `vendor/godot/{minor}/` snapshot, the regenerated `gdscript-api` data + hover index, and the `deltas/*` artifacts.
- **PR body** embeds the human changelog (step 3) + the breaking/additive classification + the test summary.
- **Add a `CHANGELOG.md` entry** (Keep a Changelog 1.1.0 headings) classifying the change as breaking vs additive for *our* consumers — distinct from Godot's own changelog.
- **Labels:** `godot-sync`, `dependencies`, plus `breaking` when step 5 flagged removals/signature changes.

### 7. TRIAGE (human — the maintainer)

The bot drafts; the maintainer decides. Specifically the maintainer:

- Reviews **Removed / Changed-signature** entries — these are the potential consumer-visible breaks. Decides whether to gate the affected symbols with `before_api` (keep them for older bundled versions) rather than delete outright.
- Decides **bundle/version policy**: add the new minor to `supported_minors`? Bump `default_minor`? Drop an EOL minor?
- Reviews the **hand-authored layer checklist** (next section) if this is a new minor — the bot cannot author annotations/keywords.
- Merges, then cuts a release (the `gdscript-api` change rides our normal release-plz flow; see `01-ARCHITECTURE.md` versioning).

## The automated bot (GitHub Action) — a concrete sketch

`.github/workflows/godot-api-sync.yml`. Implements pipeline steps 1–6. Mechanics + caveats per [`research/07`](research/07-ecosystem-and-release-tooling.md) §8.

**Caveats (load-bearing):**
- **`schedule` runs only on the latest commit of the default branch**, has a **5-minute minimum**, is **delayed under load** (schedule off-peak, e.g. `:17` not `:00`), and is **auto-disabled after 60 days of repo inactivity**.
- The default `GITHUB_TOKEN` is read-only since Feb 2023 — elevate via `permissions:`.
- **PRs opened by the default `GITHUB_TOKEN` don't trigger downstream CI.** If the sync PR must run CI (it should — step 5), use a **PAT or GitHub App token**.
- Requires the repo setting **Settings → Actions → General → "Allow GitHub Actions to create and approve pull requests."**

```yaml
# .github/workflows/godot-api-sync.yml
name: Sync Godot extension_api.json

on:
  schedule:
    - cron: '17 6 * * *'        # daily 06:17 UTC (off-peak); default-branch only; auto-disabled after 60d inactivity
  workflow_dispatch:
    inputs:
      godot_tag:
        description: 'Override Godot tag (e.g. 4.6-stable). Empty = latest stable.'
        required: false
        type: string

permissions:
  contents: write
  pull-requests: write

concurrency:
  group: godot-api-sync
  cancel-in-progress: false

jobs:
  sync:
    runs-on: ubuntu-latest
    env:
      GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
    steps:
      - uses: actions/checkout@v7

      - name: Resolve latest Godot stable tag       # PIPELINE STEP 1 (detect)
        id: godot
        run: |
          set -euo pipefail
          if [ -n "${{ inputs.godot_tag }}" ]; then
            TAG="${{ inputs.godot_tag }}"
          else
            TAG="$(gh release list -R godotengine/godot --exclude-pre-releases --exclude-drafts \
                     --limit 1 --json tagName --jq '.[0].tagName')"
          fi
          echo "$TAG" | grep -Eq '^[0-9]+\.[0-9]+(\.[0-9]+)?-stable$' || { echo "::error::bad tag $TAG"; exit 1; }
          VER="${TAG%-stable}"; MINOR="$(echo "$VER" | cut -d. -f1,2)"
          echo "tag=$TAG"     >> "$GITHUB_OUTPUT"
          echo "ver=$VER"     >> "$GITHUB_OUTPUT"
          echo "minor=$MINOR" >> "$GITHUB_OUTPUT"

      - name: Fetch extension_api.json (+ fallbacks) # PIPELINE STEP 2 (obtain)
        id: fetch
        run: |
          set -euo pipefail
          VER="${{ steps.godot.outputs.ver }}"; MINOR="${{ steps.godot.outputs.minor }}"
          P="https://raw.githubusercontent.com/godotengine/godot-cpp/godot-${VER}-stable/gdextension/extension_api.json"
          S="https://raw.githubusercontent.com/godot-rust/godot4-prebuilt/${VER}/res/extension_api.json"
          B="https://raw.githubusercontent.com/godotengine/godot-cpp/${MINOR}/gdextension/extension_api.json"
          curl -fsSL "$P" -o new_api.json || curl -fsSL "$S" -o new_api.json || curl -fsSL "$B" -o new_api.json
          # Last resort (commented): download editor binary + ./Godot --headless --dump-extension-api --dump-extension-api-with-docs
          # Fetch doc XML at tag ${VER}-stable: doc/classes/*.xml, doc/classes/@GlobalScope.xml,
          #   modules/gdscript/doc_classes/@GDScript.xml  (sparse-checkout or tarball)

      - name: Diff + classify                        # PIPELINE STEP 3 (diff)
        id: diff
        run: cargo xtask api-diff --new new_api.json --minor "${{ steps.godot.outputs.minor }}"
        # writes vendor/godot/{minor}/, vendor/godot/deltas/{old}__{new}.{json,md};
        # sets outputs: changed=<bool>, breaking=<bool>, summary=<markdown human changelog>

      - name: Regenerate gdscript-api + hover index  # PIPELINE STEP 4 (regenerate)
        if: steps.diff.outputs.changed == 'true'
        run: cargo xtask codegen-api

      - name: Test against golden fixtures            # PIPELINE STEP 5 (test)
        if: steps.diff.outputs.changed == 'true'
        run: cargo test --workspace -- --include-ignored godot_api

      - name: Open / update PR                        # PIPELINE STEP 6 (surface)
        if: steps.diff.outputs.changed == 'true'
        uses: peter-evans/create-pull-request@v8
        with:
          token: ${{ secrets.GODOT_SYNC_TOKEN }}      # PAT/App token so the PR triggers CI
          branch: chore/godot-api-sync                # stable branch => updates existing PR
          base: master
          commit-message: "chore(api): sync to Godot ${{ steps.godot.outputs.tag }}"
          title: "chore(api): sync to Godot ${{ steps.godot.outputs.tag }}"
          body: |
            Automated sync to **Godot ${{ steps.godot.outputs.tag }}** (source: godot-cpp @ same tag).
            **Breaking:** ${{ steps.diff.outputs.breaking }}
            ## API delta
            ${{ steps.diff.outputs.summary }}
          labels: |
            godot-sync
            dependencies
          delete-branch: false
```

Pinned: `actions/checkout@v7`, `peter-evans/create-pull-request@v8`. The two `cargo xtask` steps (`api-diff`, `codegen-api`) are the analyzer-specific glue defined in Phase 0; everything else is GitHub-native. A new-minor (`steps.diff` detecting `minor ∉ supported_minors`) is surfaced in the PR for the maintainer to confirm in triage.

## Versioning & multi-version policy

### Bundling multiple Godot API versions

We **bundle one API snapshot per supported Godot minor** (`bundle.supported_minors`), each under `vendor/godot/{minor}/`: that version's `extension_api.json` + doc XML + the hand-authored GDScript-layer tables + a `meta.json`. Snapshots are pulled from godot-cpp `godot-{x.y}-stable` tags (or generated). The `gdscript-api` crate is codegen'd to embed **all** bundled minors, each symbol carrying `since_api`/`before_api` gates.

### How the consumer selects a version

Detection from `project.godot` (full logic in [`research/03`](research/03-godot-api-sync.md) §6), in priority order:

1. **`[application] config/features = PackedStringArray("4.3", "Forward Plus")`** — authoritative. **Scan the array for the entry matching `\d+\.\d+`** (replicate the engine heuristic — do **not** assume index 0; a project may carry only `"4.6"`).
2. `config_version=N` (top line) — coarse era only (`5` = Godot 4.x). Not a minor signal.
3. `.gdextension` `compatibility_minimum` (4.1+) — a **floor**, not the target.

**Selection rule:** detected `{minor}` → **exact bundled match preferred**; else **snap down** to the highest bundled minor ≤ detected (additive API is a superset; watch removals/renames); else **default to the newest bundled** (`bundle.default_minor`) when `config/features` is missing/malformed. An explicit **analyzer-setting override** can force any bundled minor (for stale `config/features` or cross-target validation), mirroring godot-rust's `api-4-x` / `api-custom`. Project-local symbols (autoloads, `class_name`) are layered on top of whichever snapshot is selected.

### Deprecation policy when Godot removes API

When a diff shows a **Removed** symbol: do **not** delete it from `gdscript-api` outright. Gate it with **`before_api = "{ver}"`** so it remains available to projects targeting older bundled minors and *disappears* for projects on the newer one. Surface a `deprecated`/`removed-in-{ver}` marker for hover/diagnostics. Only physically drop a symbol when we **drop the EOL minor** from `supported_minors` (a maintainer decision).

### Support matrix (our release ↔ Godot versions covered)

Maps each analyzer release line to the Godot minors it bundles. **Illustrative — the maintainer updates this table in step 7 whenever `supported_minors`/`default_minor` change; it is the public statement of what an installed analyzer version understands.**

| Our release | Godot minors bundled | Default (unknown project) | Notes |
|---|---|---|---|
| `0.1.x` (MVP) | `4.5`, `4.6`, `4.7` | `4.7` | bootstrap set: current stable ± recent |
| `0.2.x` | `4.3`–`4.7` | `4.7` | widen back-coverage |
| `0.x` (ongoing) | `4.0`–latest stable | newest stable | add each new minor on release; carry old via `before_api` |
| `1.0.x` | `4.0`–latest stable | newest stable | EOL-drop policy starts; matrix published in docs |

Our package version (SemVer, single shared number across crates + npm — `01-ARCHITECTURE.md` versioning) is **independent of Godot's version**: a Godot-sync that only *adds* API is a patch/feature bump; one that drops a bundled minor (consumer-visible) is the 0.x "breaking" bump.

## The hand-authored GDScript layer sync (separate checklist)

The dump omits the entire GDScript *language* layer; these are **maintained by hand** in `gdscript-api` and tracked here. Run this checklist **once per Godot minor** (and whenever a release note mentions GDScript). Watch these upstream sources:

- **Annotations** — `modules/gdscript/gdscript.cpp` (the `register_annotation(...)` calls) + `modules/gdscript/doc_classes/@GDScript.xml` `<annotations>`. The 36 known: `@export*` family (`@export`, `@export_range`, `@export_enum`, `@export_flags`, `@export_file`, `@export_dir`, `@export_global_file`, `@export_global_dir`, `@export_multiline`, `@export_placeholder`, `@export_color_no_alpha`, `@export_node_path`, `@export_group`, `@export_subgroup`, `@export_category`, `@export_custom`, `@export_storage`, `@export_tool_button`, …), `@onready`, `@tool`, `@icon`, `@static_unload`, `@abstract`, `@warning_ignore`/`@warning_ignore_start`/`@warning_ignore_restore`, `@rpc`.
- **GDScript builtins** — `@GDScript.xml` `<methods>`: `preload`, `load`, `range`, `len`, `char`, `ord`, `convert`, `dict_to_inst`, `inst_to_dict`, `get_stack`, `print_debug`, `print_stack`, `type_exists`, `is_instance_of`, `Color8`, `assert`. (**`preload`/`load` appear nowhere in the dump.**)
- **Keywords** — `modules/gdscript/gdscript_tokenizer.cpp`/`.h` keyword table (and the GDScript reference). Watch for new keywords (e.g. `when`, `static`).
- **Pseudo-classes** — `@GlobalScope.xml` + `@GDScript.xml`: docs-only, never in `classes`. Re-fetch + re-render BBCode→Markdown.
- **Warning list** — `modules/gdscript/gdscript_warning.h` (the warning enum). Feeds the GDScript warning *diagnostics*, not completions; tracked here because it's a hand-authored GDScript concern that changes per minor.

**Per-minor checklist (paste into the new-minor tracking issue):**

```
[ ] Diff @GDScript.xml <annotations> vs. our annotation table  → add/rename/remove
[ ] Diff @GDScript.xml <methods> vs. our builtins table        → add/rename/remove
[ ] Diff gdscript_tokenizer keyword table vs. our keyword set  → new keywords?
[ ] Re-fetch + re-render @GlobalScope.xml / @GDScript.xml docs (BBCode→MD)
[ ] Diff gdscript_warning.h vs. our warning enum               → new/removed warnings
[ ] Update the hand-authored tables for the new {minor}, gated since_api/before_api
[ ] Add golden assertions for any new annotation/keyword/builtin
```

The Godot-sync bot **auto-fetches** `@GDScript.xml`/`@GlobalScope.xml` (so the raw input is in the PR) but **cannot author** the keyword/annotation/warning tables — that is the human's job in triage (pipeline step 7).

## Detecting GDScript LANGUAGE (not just API) changes

A **different track** from the API-data sync. New *syntax* (not new symbols) never appears in `extension_api.json` — it changes the **grammar**, so it routes to the **parser/HIR backlog**, not the codegen pipeline. Examples of past language changes: typed dictionaries `Dictionary[K, V]` (4.4), `static var`, `@abstract` (4.5), lambdas, typed arrays.

**How we notice (in order of usefulness):**

1. **Release notes / blog** — `https://godotengine.org/releases/{minor}/` and the "Godot {x.y} is here" articles; the **GDScript** subsystem section of `CHANGELOG.md` (per release branch).
2. **The Interactive Changelog** — `https://godotengine.github.io/godot-interactive-changelog/` (filter to the `gdscript` module / PRs labeled `topic:gdscript`).
3. **Doc diffs** — changes to `@GDScript.xml`, the GDScript reference page, and the tokenizer/parser source under `modules/gdscript/`.
4. **The hand-authored-layer checklist** above will surface new *keywords* (a strong syntax-change signal) even if we miss the release note.

**Routing:** when a language change is detected, open a **parser/HIR backlog issue** (label `lang-change`, link the PR/changelog), *not* a Godot-sync PR. The Godot-sync pipeline (this doc) handles **data**; grammar/inference work is Phase 1–2 territory ([`01-ARCHITECTURE.md`](01-ARCHITECTURE.md) §6, [`research/04`](research/04-gdscript-semantics-and-features.md), [`research/09`](research/09-type-system-and-inference.md)). Keeping these tracks separate is deliberate: a data sync is mechanical and auto-mergeable; a grammar change is design work.

## Failure modes & manual fallback

| Failure | Symptom | Fallback |
|---|---|---|
| **No prebuilt dump yet** (godot-cpp tag lags Godot stable) | `fetch.api_json.primary` 404s | try `secondary` (godot4-prebuilt), then `branch`/`master`; if all stale, **generate** from the editor binary (`generate_fallback`). The bot retries on the next cron once the tag lands. |
| **godot-cpp lags persistently** | primary keeps 404ing for days | run `generate_fallback` in CI (download `Godot_v{ver}-stable_linux.x86_64.zip` from `downloads.godotengine.org`, `--headless --dump-extension-api …`). Pin the binary checksum. |
| **Malformed / truncated dump** | JSON parse error, or `header.version_full_name` mismatches the tag, or size wildly off (<5 MB) | **abort the PR**, open an issue, fall back to the secondary source or generate. The permissive raw-JSON layer tolerates *unknown keys* but not invalid JSON — validate `header` + size before diffing. |
| **`hash` noise from precision** | every method shows a `hash` delta | confirm both dumps are `precision: single` (our `hash_scope`); a double-precision dump produces different hashes — discard and re-fetch the single-precision artifact. |
| **No release detected for a long time** | cron silent | the **60-day auto-disable** may have fired — a repo-activity commit re-enables it; also test via `workflow_dispatch`. |
| **PR doesn't trigger CI** | sync PR sits with no checks | the default `GITHUB_TOKEN` was used — switch to the PAT/App token (`GODOT_SYNC_TOKEN`). |
| **Hand-authored layer drift** | annotations/keywords stale after a minor | the per-minor checklist was skipped — it is a *manual* gate; the bot only fetches the XML, it cannot author the tables. |

When fully manual: a maintainer runs the same pipeline by hand — fetch the dump from a [version→URL row](research/03-godot-api-sync.md#55-version--url-table) in `research/03`, run `cargo xtask api-diff` then `cargo xtask codegen-api` locally, run the tests, and open the PR. The runbook is identical; only the trigger differs.

## Exit criteria / definition of "synced"

The analyzer is **synced to Godot `{ver}`** when **all** hold:

1. `vendor/godot/{minor}/extension_api.json` has `header.version_full_name == "Godot Engine v{ver}.stable.official"` and the recorded `precision` matches `hash_scope`.
2. `cargo xtask codegen-api` regenerated `gdscript-api` from that snapshot with no errors; `since_api`/`before_api` gates are set.
3. The doc-XML hover index for `{minor}` is regenerated (no literal `[bbcode]` leaks into hover).
4. The hand-authored GDScript layer passed its per-minor checklist (annotations/keywords/builtins/warnings current for `{minor}`).
5. The golden-fixture suite passes against the snapshot (engine-symbol assertions green).
6. The API-delta (`deltas/{old}__{new}.{json,md}`) is committed, and any **Removed/Changed-signature** entries were triaged (gated with `before_api`, not silently dropped).
7. `bundle.supported_minors` (and `default_minor` if newest) reflect the new minor; the support-matrix table is updated.
8. A `CHANGELOG.md` entry classifies the change (breaking vs additive) for our consumers, and the sync PR is merged.

"Synced" is **per bundled minor**: each entry in `supported_minors` independently satisfies 1–6; the bundle as a whole satisfies 7–8.
